# Money Review — UI設計書

## 1. デザイン方針

| 項目 | 方針 |
|------|------|
| レイアウト | モバイルファースト（375px基準）、Mac では最大600px幅でセンタリング |
| カラー | ホワイト背景 + アクセントカラー1色（インディゴ `#4F46E5`） |
| フォント | システムフォント（`-apple-system, BlinkMacSystemFont, "Segoe UI"`） |
| タッチターゲット | 最小44×44px |
| アニメーション | トランジション0.15s以内、必要最小限 |

---

## 2. レイアウト構造

```
┌─────────────────────────────────────────┐
│  ヘッダー                                │
│  [アプリ名]           [月セレクト ▼]      │
├─────────────────────────────────────────┤
│                                         │
│  コンテンツエリア（タブで切り替え）         │
│                                         │
│                                         │
├─────────────────────────────────────────┤
│  ボトムナビゲーション                     │
│  [ホーム] [登録] [一覧] [カテゴリ]        │
└─────────────────────────────────────────┘
```

### ヘッダー

- 高さ: 56px
- 左: アプリ名「Money Review」（16px、medium）
- 右: 月セレクトボックス（`YYYY年MM月`形式）

### ボトムナビゲーション

- 高さ: 56px + safe-area-inset-bottom
- タブ: ダッシュボード / 登録 / 一覧 / カテゴリ
- アクティブ: アクセントカラー + アイコン塗りつぶし
- 非アクティブ: グレー

## 3. 画面設計

### 3-1. ダッシュボード

```
┌─────────────────────────────────────────┐
│  今月の支出                               │
│  ¥53,800                                │
├───────────────────────────────────────  │
│  先月比較                                 │
│  今月 ¥53,800   先月 ¥48,200             │
│  差額 +¥5,600（+11.6%）↑                │
├─────────────────────────────────────────┤
│  カテゴリ別支出                           │
│  [円グラフ]                              │
│  ■食費  ■外食  ■趣味  □その他            │
├─────────────────────────────────────────┤
│  支出ランキング                           │
│  1位 食費        ¥25,000  46.5%         │
│  2位 外食        ¥12,000  22.3%         │
│  3位 趣味         ¥8,000  14.9%         │
├─────────────────────────────────────────┤
│  今月のメモ                              │
│  今月は外食が多かった。                   │
│  来月は食費を抑えたい。         [編集]    │
└─────────────────────────────────────────┘
```

#### 先月比較カード

- 増加: 差額・増減率を赤文字 + ↑
- 減少: 差額・増減率を緑文字 + ↓
- 先月データなし: 「先月データなし」と表示

#### 円グラフ

- ライブラリ: Recharts（`PieChart`）
- 中央に総支出額を表示（ドーナツグラフ）
- 凡例: グラフ下部に横並び、5件まで表示（それ以上は「その他」に統合）
- タップで該当カテゴリをハイライト

#### ランキング

- 最大5位まで表示
- 各行: 順位 | カテゴリ名 | 金額 | 割合バー

### 3-2. 支出登録

```
┌─────────────────────────────────────────┐
│  日付                                    │
│  [2026-06-01          ▼]                │
├─────────────────────────────────────────┤
│  支出を入力（1行 = 1件）                  │
│  ┌──────────────────────────────────┐   │
│  │ 食費 5000 オーケーストア           │   │
│  │ 外食 1200 ラーメン               │   │
│  │ 趣味 3000 書籍                   │   │
│  │                                  │   │
│  └──────────────────────────────────┘   │
│  形式: カテゴリ 金額 内容 [メモ]          │
├─────────────────────────────────────────┤
│  入力例                                  │
│  食費 5000 スーパー                      │
│  外食 1200 ランチ 友人と               │
├─────────────────────────────────────────┤
│  エラー表示エリア（エラー時のみ表示）      │
│  ⚠ 行2: カテゴリ「外食」は存在しません   │
├─────────────────────────────────────────┤
│  [  一括登録（3件）  ]                   │
└─────────────────────────────────────────┘
```

#### テキストエリア

- 最小高さ: 160px、最大: 60vhまで自動拡張
- フォント: モノスペース（`font-mono`）16px
- プレースホルダー: `食費 5000 スーパー\n外食 1200 ランチ`

#### リアルタイムバリデーション

- テキスト変更 300ms debounce 後にパース実行
- エラー行: 行番号付きで一覧表示
- 登録ボタン: エラーがあれば `disabled`

#### 一括登録ボタン

- テキスト: `一括登録（N件）`（正常パース行数を表示）
- 登録中: `登録中...` + スピナー
- 成功後: フォームクリア + 成功トースト

### 3-3. 支出一覧

```
┌─────────────────────────────────────────┐
│  [月セレクト]    合計 ¥53,800  全N件     │
├─────────────────────────────────────────┤
│  06/01 食費     ¥5,000  オーケーストア   │
│                                [編集][削]│
├─────────────────────────────────────────┤
│  06/01 外食     ¥1,200  ラーメン         │
├─────────────────────────────────────────┤
│  （編集展開時）                          │
│  ┌──────────────────────────────────┐   │
│  │ 日付: [2026-06-01]               │   │
│  │ カテゴリ: [食費      ▼]           │   │
│  │ 金額: [5000        ]             │   │
│  │ 内容: [オーケーストア]             │   │
│  │ メモ: [            ]             │   │
│  │    [キャンセル]    [保存]          │   │
│  └──────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

#### リスト行

- 日付（MM/DD）| カテゴリバッジ | 金額（右寄せ）| 内容
- 右端: 編集アイコン / 削除アイコン
- 削除: 確認ダイアログ（`window.confirm` or カスタムモーダル）

#### 編集フォーム

- 行をタップまたは編集アイコンタップで展開
- インラインで展開（アコーディオン）
- カテゴリ: `<select>` でカテゴリ一覧
- 日付: `<input type="date">`

#### ソート・フィルタ

- デフォルト: 日付降順
- MVP ではソート変更不要

## 5. 共通コンポーネント仕様

### LoadingSpinner

```
サイズ: 24px（inline用）/ 48px（全画面用）
色: アクセントカラー
```

### Toast

```
位置: 画面上部中央、固定（z-index: 1000）
種類:
  success: 緑背景
  error:   赤背景
  info:    グレー背景
表示時間: 3秒後に自動消去
アニメーション: スライドイン（上から）
```

### ConfirmDialog

```
タイトル + メッセージ + [キャンセル][確認]
モーダル表示（backdrop: rgba(0,0,0,0.4)）
```

### カテゴリバッジ

```
小さい角丸ピル型
カテゴリごとに色は固定（最初の5色をプリセット、追加分はグレー）
```

## 6. レスポンシブ対応

| ブレークポイント | 対応 |
|---------------|------|
| 〜 480px（iPhone） | シングルカラム、ボトムナビ |
| 481px 〜（Mac） | 最大幅 600px、左右マージン auto |

Mac では画面幅に余裕があるため：
- 支出一覧: テーブル形式（ヘッダー行あり）に変更
- ダッシュボード: 先月比較とランキングを横並びに変更

---

## 7. フォームの入力補助

### 登録画面の日付

- デフォルト: 今日の日付
- 変更: `<input type="date">`（iOSネイティブピッカーを活用）

### カテゴリ入力補助

登録テキストエリアでのカテゴリ入力を支援するため：
- テキストエリア上部に現在のカテゴリ一覧をタップ挿入ボタンとして表示（任意実装）

---

## 8. 空状態（Empty State）

| 画面 | 表示内容 |
|------|---------|
| ダッシュボード（データなし） | 「今月の支出はまだありません。登録タブから入力してください」 |
| 支出一覧（データなし） | 「この月の支出はありません」 |
| カテゴリ（0件） | 「カテゴリを追加してください」 |
